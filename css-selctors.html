Selektory i specyficzność

Selektory

Selektor to reguła, która mówi przeglądarce, jakie elementy drzewa DOM aplikacji webowej powinna zaznaczyć.

  • * → zaznacza wszystkie elementy drzewa DOM aplikacji webowej.

    Przykład> 
     **Kod css:**
 ```css
 * {
     colore: red;
 }
 ```
 **Kod html:**
 ```html
 <ul>
     <li>I am red</li>
     <li>I am red</li>
     <li>I am red</li>
 </ul>
 ```
    Wynik
    • I am red
    • I am red
    • I am red

  • li → zaznacza wszystkie zdefiniowane elementy np.: li

    Przykład> 
     **Kod css:**
 ```css
 li {
     colore: red;
 }
 ```
 **Kod html:**
 ```html
 <ul>
     <li>I am red</li>
     <li>I am red</li>
     <li>I am red</li>
 </ul>
 <p>I am NOT red</p>
 ```
    Wynik
    • I am red
    • I am red
    • I am red

    I am NOT red

  • .className → zaznacza wszystkie elementy o zdefiniowanej nazwie klasy

    Przykład> 
     **Kod css:**
 ```css
 .list--item__middle-item {
     colore: red;
 }
 ```
 **Kod html:**
 ```html
 <ul>
     <li>I am NOT red</li>
     <li class="list--item__middle-item">I am red</li>
     <li>I am NOT red</li>
 </ul>
 ```
    Wynik
    • I am NOT red
    • I am red
    • I am NOT red

  • #id → zaznacza element o zdefiniowanym id

    Przykład> 
     **Kod css:**
 ```css
 #unique-lit-item {
     colore: red;
 }
 ```
 **Kod html:**
 ```html
 <ul>
     <li id="unique-lit-item">I am red</li>
     <li>I am NOT red</li>
     <li>I am NOT red</li>
 </ul>
 ```
    Wynik
    • I am red
    • I am NOT red
    • I am NOT red

  • [attr="value"] → zaznacza wszystkie elementy zawierające zdefiniowany atrybut o zdefiniowanej wartości. Aby zaznaczyć grupę atrybutów, możemy skorzystać z wyrażeń regularnych np.: [href$=".com"], zaznaczy wszystkie elementy z atrybutem href kończącym się na .com.

    Przykład> 
     **Kod css:**
 ```css
 [href$=".com"] {
     colore: red;
 }
 ```
 **Kod html:**
 ```html
 <ul>
     <li><a href="test.org">I am NOT red</a></li>
     <li><a href="test.com">I am red</a></li>
     <li><a href="test.com">I am red</a></li>
 </ul>
 ```
    Wynik
    • I am red
    • I am red
    • I am red

  • h1 + p → musi być umieszczony pomiędzy dwoma selektorami. Zaznaczy pierwszy element na tym samym poziomie hierarchii co element zaznaczony przez pierwszy selektor, który występuje zaraz PO tym elemencie.

    Przykład> 
     **Kod css:**
 ```css
 h1 + p {
     colore: red;
 }
 ```
 **Kod html:**
 ```html
 <article>
     <p>I am NOT red</p>
     <strong>Reference element</strong>
     <p>I am red</p>
 </article>
 ```
    Wynik

    I am NOT red

    Reference element

    I am red

  • h1 ~ p → musi być umieszczony pomiędzy dwoma selektorami. Zaznaczy wszystkie elementy na tym samym poziomie hierarchii co element zaznaczony przez pierwszy selektor, które występują po tym elemencie.

    Przykład> 
     **Kod css:**
 ```css
 h1 ~ p {
     colore: red;
 }
 ```
 **Kod html:**
 ```html
 <article>
     <p>I am NOT red</p>
     <strong>Reference element</strong>
     <p>I am red</p>
     <div>I am NOT red</div>
     <p>I am red</p>
 </article>
 ```
    Wynik

    I am NOT red

    Reference element

    I am red

    I am NOT red

    I am red

  • ul > li → musi być umieszczony pomiędzy dwoma selektorami. Zaznaczy wszystkie elementy, które są BEZPOŚREDNIMI dziećmi elementu zaznaczonego przez pierwszy selektor.

    Przykład> 
     **Kod css:**
 ```css
 ul > li {
     colore: red;
 }
 ```
 **Kod html:**
 ```html
 <ul>
     <li>I am red</li>
     <li>I am red</li>
     <li>I am red</li>
     <ul>
         <li>I am NOT red</li>
         <li>I am NOT red</li>
         <li>I am NOT red</li>
     </ul>
 </ul>
 ```
    Wynik
    • I am red
    • I am red
    • I am red
      • I am NOT red
      • I am NOT red
      • I am NOT red

  • ul.className li → zaznacza elementy, które wyznaczają zadeklarowaną ścieżkę hierarchii pośrednio, lub bezpośrednio.

    Przykład> 
     **Kod css:**
 ```css
 ul.list-item em {
     colore: red;
 }
 ```
 **Kod html:**
 ```html
 <ul>
     <li class="list-item">I am NOT red</li>
     <li class="list-item">I am NOT red</li>
     <li class="list-item"><em>I am red</em></li>
 </ul>
 ```
    Wynik
    • I am red
    • I am red
    • I am red

Specyficzności

Jest to ważność naszej reguły względem innych reguł. Moc reguły można policzyć na podstawie zastosowanych selektorów np.:

  1. #user-name.experience li → będzie miał moc równą 111
  2. ul li → będzie miał moc 2

Wynika z tego, że selektor 1 nadpisze swoimi stylami style z selektora 2, ponieważ jest on ważniejszy.

Każdy style mogą zostać nadpisane przez klauzule !important, lecz nie zaleca się stosowania tej klauzuli, ponieważ może być ona nadpisana inną klauzulą !important o większej specyficzności.

Na specyficzność ma też wpływ, w jakiej kolejności ustawione są nazwy klas w atrybiecie class nadanym na element HTML. Każdy kolejny selektor ma większą specyficzność niż poprzedni jeżeli przyjmiemy, że wszystkie selektory zawarte w atrybucie class mają te samą specyficzność.

Moc specificzności selektorów opisuje poniższa tabelka:

Selektory Moc
h1, [attribute=“value”], :pseudo-class, ::pseudo-element 1
.class 10
#id 100
inline style 1000

Selektory: *, +, ~, > oraz pseudo klasa :not nie mają wpływu na specyficzność całego wyrażenia selektora

Warto wiedzieć

Do stylowania elementów nie powinno używać się id’ków (#), ponieważ ten selektor ma bardzo dużą moc i aby nadpisać style dla elementów zawartych w elemencie ostylowanym id’ikmie należałoby użyć co najmniej 10 klas, lub kolejnego id co czyni nasz kod css nieutrzymywalnym
